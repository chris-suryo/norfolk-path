# Level design — how the world gets built

This is the shared contract between the **story session** (owns the vision —
what places exist, who lives there, how the level *feels*) and the **build
session** (owns the geometry — turning that vision into a valid, playable map).
Read this before writing a brief or editing the map.

## The loop

```
story session writes docs/world-brief.md   (the vision: zones, cast, pacing)
        │
        ▼
build session edits scripts/island_map.gd  (translates brief → ASCII map)
        │
        ▼
python tools/preview_map.py  →  docs/island-preview.png   (see it + validate it)
        │
        ▼
review the image together → refine the brief → repeat
```

The level is **not** generated by an image model. A diffusion/image generator
makes a flat picture you can't walk in — no tile grid, no collision, no link to
the tileset. It's fine for *mood boards*, useless as a *level*. Cohesion comes
from the story driving the geography, then fast iteration on the ASCII map with
the preview render — not from one-shotting an image.

## The map

The whole level is an ASCII grid in `scripts/island_map.gd` (the `MAP` const).
One character = one 16px tile. The current map is **70 wide × 45 tall**.
`tools/preview_map.py` reads it directly and renders `docs/island-preview.png`.

## The palette — what art we actually have

Design **within this set** (Cute Fantasy Free pack). Asking for something not
here means it can't be built without sourcing new art.

- **Ground:** grass, dirt path, water, farmland (tilled dirt fields). *(Still
  unused: beach/sand and cliff — sand's art is for water edges not grass, and
  cliffs imply elevation, so both need a dedicated pass.)* Farmland fields must
  be **solid rectangles ≥2 wide** — its 3×3 tile sheet can't tile a field with
  prop-holes punched in it, so put crops/animals on grass beside the field, not
  on the dirt.
- **Nature:** oak trees (big + small), rocks, tree stumps, wildflowers, potted
  garden flowers, carrot/veg plants. *(No round bushes — the pack's "bushes"
  are tiny sprigs that vanish on grass, so they're dropped.)*
- **Structures:** one house sprite (blue roof — reused for both the shop and the
  library), wood fences, a wood bridge, a lamp post, a chest. *(No shop
  interiors, wells, market stalls, or castle/dungeon pieces.)*
- **Creatures:** chicken, cow, pig, sheep (farm animals); slime + skeleton
  (enemies, unused so far). *(No duck yet — Ariana is a chicken stand-in.)*
- **People:** one player sprite (reused for both players and — for now — any
  NPC would reuse it too, recolored later).

If the story needs a place we have no art for (a cave, a tavern interior, a
market), flag it in the brief as `[NEEDS ART]` — we decide whether to source it,
fake it with what we have, or cut it. Don't silently assume it exists.

## Hard rules (the geometry constraints)

1. **Water and path regions must be ≥ 2 tiles wide.** The edge/shoreline art
   has corners and edges but **no 1-tile strip pieces**, so a 1-wide channel
   can't render. `tools/preview_map.py` flags violations.
2. **Keep the map rectangular** — every row the same width.
3. **One level → one boss** (v1 scope). This is a single connected island, not
   a multi-map world. Don't design a second zone without raising scope first.
4. **Spawn (`S`) on path; shop (`H`) and chicken (`C`) on grass.** The validator
   checks this.
5. Entities and props draw with **y-sorting** — things lower on screen overlap
   things higher, so the player walks in front of a tree's base and behind its
   top. Place tall props (trees, house) where that reads naturally.

## How to write a brief

The story session fills in `docs/world-brief.md`. You describe **intent**, not
tiles — the build session turns intent into the grid. For each zone, say:

- **Name & purpose** — what this place is, why the player is here.
- **Who / what is here** — NPCs, the shop, animals, obstacles.
- **Connections** — which zones it borders and how you get between them
  (path? bridge? a gap in the trees?).
- **Feeling / pacing** — open and safe, dense and tense, a quiet corner? Where
  does it sit in the arc (early/mid/approach-to-boss)?

Plus, once, up top: the **overall arc** (spawn → … → boss) and the **cast**
(who each NPC is and roughly where they belong).

## Worked example — the current island, in brief form

This is the map that exists today, written the way a brief would describe it —
use it as a model for the shape and level of detail.

> **Arc:** spawn at the west end of a path → follow it east past Evan's shop →
> cross a bridge over a stream → reach the pond where Ariana lives. (No boss
> placed yet.)
>
> **Zone — West path (spawn):** open grass, the path begins here; both players
> start side by side. Calm, an easy on-ramp. Connects east along the path.
>
> **Zone — Evan's shop:** a house set back north of the path, framed by a fence
> yard with a gap for the door. A path spur leads up to it. Walking to the door
> shows Evan's line ("Toasted, or the old way?"). The social hub of the west
> side.
>
> **Zone — The stream & bridge:** a stream cuts north–south across the middle;
> the path crosses it on a wood bridge — the one pinch-point between west and
> east. Gives the island a sense of two halves.
>
> **Zone — Ariana's pond (east):** the path ends at a pond/bay in the east.
> Ariana (a chicken for now, should be a duck) is at the water's edge. Trees
> cluster around the southern shore. Quiet, a natural destination.

## The round-trip mechanics

- **Story session:** write/refine `docs/world-brief.md`. If a Claude Code
  session on this repo, you can also run `python tools/preview_map.py` to see
  the *current* map before briefing changes.
- **Build session:** edit `scripts/island_map.gd`, run `python
  tools/preview_map.py` (regenerates `docs/island-preview.png`, must report **0
  problems**), eyeball the render, then commit. Chris verifies in-engine.
- `tools/preview_map.py` mirrors the engine's autotiling and y-sort, so the
  preview is faithful — but it is **not** the engine. Final truth is Chris's
  Windows import/export/playtest.
